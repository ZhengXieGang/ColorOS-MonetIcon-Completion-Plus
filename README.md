## 使用方法
1. 下载release里的webui.zip。
2. 将webui.zip的内容放入原模块的目录`data/adb/modules/ThemedIconCompletion`或放入模块文件的压缩包内刷入。
3. 更新完图标记得在管理器里关掉系统桌面的默认卸载模块，重新更换一次图标包。

## 适配指南

1.  **准备环境**
    创建一个文件夹，将仓库内的两个py脚本放入，同时在文件夹内创建名为 `icon` 的文件夹，放置原始图标文件（**注意**：需将原图标文件名更改成**包名**，文件后缀无所谓）。

2.  **生成目录**
    运行 `mkdir.py`。
    * 脚本会自动根据 `icon` 文件夹内的文件名，在当前目录下创建 `uxicons` 文件夹并创建包名目录。

3.  **处理图标**
    将绘制好的图标放入 `uxicons` 内对应的文件夹后，执行 `process_icons.py`。
    * 脚本会将图标转换成RGBA格式并清除图片元数据。
    * 脚本会将 `uxicons` 文件夹内所有文件重命名为 `monochrome.png`。
    
4.  **提交请求**
    * 
    * Frok本仓库，将本地`uxicon`文件夹中的内容上传到自己的`uxicon`文件夹中。
    * 创建pull request,合并后会自动Release新版本。

## 图标规范

* **参考文件**：图标可参考 `icons.psd`。
* **尺寸标准**：画布大小为 **240x240** 像素，图标本体取最长一边，调整为 **90** 像素后居中（可按图标实际情况进行适当修改）。
* **颜色原理**：莫奈图标本质上是一层**遮罩**。
    * 颜色（灰度/明度）取决于图标该处的**透明度**，而并非颜色本身。
    * 可通过调整图标部分区域的透明度来实现不同的灰度。

## 提出适配需求

1.  找到你想要求适配的应用的安装包。
2.  打开后找到 `res` 文件夹解压出来。
3.  将 `res` 文件夹重命名为对应的**应用包名**。
4.  将所有文件夹放到一个压缩包内并提交 Issue。

> **注意**：提交适配前请自行检查，确保该图标未被该项目适配。
